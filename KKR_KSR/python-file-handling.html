Module 1

Basics of File Handling

What is a File?

A file is a named collection of data stored on disk. Without file handling, data exists only while your program is running. Files let you save information permanently.

project structure 
project/
│
├── main.py          # Python code
├── data.txt         # Plain text
├── students.csv     # Tabular data
├── config.json      # Configuration data
└── image.png        # Binary image data

Opening a File — open()

Python provides the built-in open() function. The variable file is a file object that lets you read or write.

opening a file 
# Syntax
file = open(filename, mode)

# Example — read mode
file = open("data.txt", "r")

Closing a File

After finishing with a file, close it to flush buffered data and release OS resources.

manual close 
file = open("data.txt", "r")

# ... work with file ...

file.close()

Using with ✅ Recommended

Python automatically closes the file when the with block ends — even if an exception occurs.

context manager 
with open("data.txt", "r") as file:
    data = file.read()

print(data)
⚠️

Common mistakes: forgetting to close the file, using "w" when you only want to read (erases the file!), or opening a non-existent file in "r" mode — raises FileNotFoundError.

Module 2

Reading Files

💡

Think of the file as a long sequence of characters. Python keeps an invisible cursor (file pointer) that moves forward as you read.

read() — Read Entire File

read entire file 
with open("students.txt", "r") as file:
    data = file.read()
    print(data)
Output
Alice
Bob
Charlie
David
Emma
⚠️

Calling read() a second time returns an empty string '' — the cursor is already at EOF.

read(size) — Read N Characters

read in chunks 
with open("students.txt", "r") as file:
    print(file.read(5))   # → "Alice"
    print(file.read(5))   # → "\nBob\n"
    print(file.read(5))   # → "Charl"

Notice: \n is also a character — read(size) crosses line boundaries.

readline() — One Line

readline 
with open("students.txt", "r") as f:
    print(f.readline().strip())
    print(f.readline().strip())
Output
Alice
Bob

readlines() — List of Lines

readlines 
with open("students.txt", "r") as f:
    lines = f.readlines()
    print(lines)
Output
['Alice\n', 'Bob\n', 'Charlie\n',
 'David\n', 'Emma']

for loop ✅ Best for Large Files

Processes one line at a time — does not load the whole file into memory.

for loop iteration 
with open("students.txt", "r") as file:
    for line in file:
        print(line.strip())

Module 3

Writing Files

write() — Write a String

Opening in "w" mode immediately empties the file before you write anything.

⚠️ Loses newlines
with open("f.txt", "w") as f:
    f.write("Alice")
    f.write("Bob")
# Result: AliceBob
✅ Correct
with open("f.txt", "w") as f:
    f.write("Alice\n")
    f.write("Bob\n")
# Result: Alice / Bob

writelines() — Write a List

writelines 
students = [
    "Alice\n",
    "Bob\n",
    "Charlie\n"
]

with open("s.txt", "w") as f:
    f.writelines(students)
⚠️

writelines() does not add \n automatically. You must include it yourself.

Append Mode (a)

append 
# Adds to end, nothing deleted
with open("s.txt", "a") as f:
    f.write("David\n")
    f.write("Emma\n")
Alice Bob Charlie David ← cursor

Create Mode (x) — Exclusive Create

Creates a new file. Raises FileExistsError if the file already exists — protects you from accidental overwrites.

exclusive create 
with open("report.txt", "x") as f:
    f.write("Python")

# Running again → FileExistsError
ModeFile ExistsDoesn't ExistOld Data

"w"OverwritesCreatesDeleted
"a"AppendsCreatesKept
"x"ErrorCreatesN/A

Module 4

File Pointer (Cursor)

🎯

Memory Trick: Think of the file pointer like a YouTube progress bar. tell() asks "where are we?" and seek(0) drags it back to the beginning.

tell() — Current Position

tell() 
with open("students.txt", "r") as f:
    print(f.tell())   # 0
    f.read(5)
    print(f.tell())   # 5
Output
0
5

seek() — Move Cursor

seek() 
with open("students.txt", "r") as f:
    f.read()         # read all
    f.seek(0)        # reset cursor
    print(f.read())  # reads again!
💡

seek(n) moves to absolute position n from start — not relative.

Reading After Writing — the Classic Trap

❌ Returns empty string
with open("f.txt", "w+") as f:
    f.write("Python")
    print(f.read())   # ''
✅ Correct — seek first
with open("f.txt", "w+") as f:
    f.write("Python")
    f.seek(0)
    print(f.read())   # Python

Complete Example — tell() + seek() + write()

combined example 
with open("sample.txt", "w+") as f:

    f.write("Hello")
    print(f.tell())    # → 5

    f.write(" World")
    print(f.tell())    # → 11

    f.seek(0)
    print(f.read())    # → Hello World
Output
5
11
Hello World
MethodPurpose
tell()Returns the current cursor position (integer)
seek(pos)Moves cursor to absolute position pos
read()Reads from the current cursor position
write()Writes at the current cursor position

Module 5

File Modes — Complete Guide

🧠

Memory trick: r = Read · w = Wipe & Write · a = Append · x = eXclusive create · + = also allow the other operation · b = Binary

r — Read Only

r mode
with open("s.txt", "r") as f:
    print(f.read())
  • File must already exist
  • Writing raises io.UnsupportedOperation
  • Cursor starts at position 0

w — Write (Wipe)

w mode
with open("s.txt", "w") as f:
    f.write("David")
  • Creates file if missing
  • Deletes all existing content immediately

r+ — Read & Overwrite

r+ mode
# File: "Python"
with open("f.txt", "r+") as f:
    f.seek(0)
    f.write("Hi")
# Result: "Hithon" (overwrites)
  • File must exist
  • Overwrites bytes at cursor position
  • Does NOT delete existing content

a+ — Append & Read

a+ mode
with open("s.txt", "a+") as f:
    f.write("\nCharlie")
    f.seek(0)
    print(f.read())
  • Writes always go to the end
  • You can seek() to read anywhere
  • Creates file if missing

Binary Modes

Used for non-text files — images, PDFs, videos, ZIP files. Returns bytes instead of str.

binary modes
# Read an image
with open("photo.png", "rb") as f:
    data = f.read()   # type: bytes

# Copy a file
with open("src.png", "rb") as src:
    with open("dst.png", "wb") as dst:
        dst.write(src.read())
ModeReadWriteCreate if MissingDeletes ExistingWrite Position
r✅❌❌❌—
w❌✅✅✅Beginning (empty)
a❌✅✅❌End
x❌✅✅❌Beginning (new file)
r+✅✅❌❌Current cursor
w+✅✅✅✅Beginning (empty)

Interview Q&A

Q: Which mode is safest if you don't want to accidentally overwrite an existing file?

→ "x" — raises FileExistsError if the file already exists.

Q: Which mode should you use for log files?

→ "a" — adds new entries without deleting old ones.

Q: Which mode deletes file contents immediately when opened?

→ "w" and "w+"

Q: Which mode lets you read and always append to the end?

→ "a+"
