1. INTRODUCTION

1.1 Purpose

The purpose of this document is to define the functional and non-functional requirements for the Hospital Management System.

The system will enable hospital administrators, doctors, and staff to manage patient records, doctor records, appointments, treatments, billing, and reporting operations efficiently.

This document serves as the primary reference for developers, testers, architects, and stakeholders.

1.2 Scope

The Hospital Management System shall provide the following modules:

* Doctor Management
* Patient Management
* Appointment Management
* Treatment Management
* Billing Management
* Reporting and Analytics
* Data Validation
* Exception Handling

The application shall be implemented using Python and Object-Oriented Programming principles.

1.3 Objectives

The objectives of the system are:

* Eliminate manual record maintenance
* Maintain centralized patient records
* Prevent duplicate appointments
* Automate billing calculations
* Improve data accuracy
* Generate operational reports

4. FUNCTIONAL REQUIREMENTS

FR-001 Add Doctor

Description

The system shall allow administrators to register a doctor.

Input Fields

Doctor ID
Name
Age
Phone Number
Specialization
Consultation Fee

Validation Rules

* Doctor ID must be unique
* Age must be greater than zero
* Consultation fee must be greater than zero
* Phone number must contain ten digits

Output

Doctor registered successfully.

---

FR-002 Search Doctor

Input

Doctor ID

Process

Search doctor records using Doctor ID.

Output

Display doctor information.

Exception

DoctorNotFoundException

---

Validation Rules

8. CUSTOM EXCEPTION REQUIREMENTS

DoctorNotFoundException

Raised when a doctor record does not exist.

PatientNotFoundException

Raised when a patient record does not exist.

AppointmentAlreadyExistsException

Raised when an appointment slot is already booked.

InvalidAgeException

Raised when age is less than or equal to zero.

InvalidFeeException

Raised when consultation fee is invalid.

BillingException

Raised when billing generation fails.
